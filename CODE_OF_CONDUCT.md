# Code of Conduct

## Our Pledge

We want DriveMind to be a respectful and welcoming project for everyone.

Contributors are expected to communicate respectfully, collaborate constructively, and help maintain a positive environment.

---

## Expected Behavior

Examples of expected behavior:

- Be respectful and professional
- Give constructive feedback
- Be open to different ideas
- Help others learn
- Focus on improving the project

---

## Unacceptable Behavior

Examples of unacceptable behavior:

- Harassment or personal attacks
- Insulting or disrespectful comments
- Discrimination of any kind
- Spamming issues or pull requests
- Sharing private information without permission

---

## Reporting Issues

If you notice behavior that violates this code of conduct, please report it through a GitHub issue or contact the project maintainer.

---

## Scope

This code of conduct applies to all project spaces, including:

- GitHub issues
- Pull requests
- Discussions
- Documentation
- Project communication

---

## Goal

The goal is to keep DriveMind collaborative, respectful, and useful for learning and innovation.
